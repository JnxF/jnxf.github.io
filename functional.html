How functional is it?

Once you learn the slightest bit about functional programming, you can no longer escape it.
How well do the programming languages manage chained higher order functions calls?

The problem

Add the salary of all the employees over 40.

employees = [
    { name: "Adam",  salary: 2000, age: 20 },
    { name: "Bob",   salary: 3000, age: 50 },
    { name: "Clair", salary: 5000, age: 60 }
]

🪜 Scala

Clean. Vibrant. Brilliant. — It shows that Scala was created with functional programming in mind.

★★★★★ 
class Employee(val name : String, val salary : Int, val age : Int)

val employees = List(
    new Employee(name = "Adam", salary = 2000, age = 20),
    new Employee(name = "Bob", salary = 3000, age = 50),
    new Employee(name = "Clair", salary = 5000, age = 60)
)

employees
    .filter(_.age > 40)
    .map(_.salary)
    .sum

🟡 Javascript

There is no sum function, but creating objects on the fly is pleasant.

★★★★☆ 
let employees = [
    { name: "Adam", salary: 2000, age: 20 },
    { name: "Bob", salary: 3000, age: 50 },
    { name: "Clair", salary: 5000, age: 60 }
]
employees
    .filter(e => e.age > 40)
    .map(e => e.salary)
    .reduce((v, acc) => v + acc)

🐍 Python

Two main flavours of functional programming. None of them is fully satisfactory, but list comprehensions come handy.

★★☆☆☆ 
import collections

Employee = collections.namedtuple(
    "Employee", ["name", "salary", "age"]
)

employees = [
    Employee(name="Adam", salary=2000, age=20),
    Employee(name="Bob", salary=3000, age=50),
    Employee(name="Clair", salary=5000, age=60),
]

sum(map(lambda e: e.salary, filter(lambda e: e.age > 40, employees)))

sum([e.salary for e in employees if e.age > 40])

💠 F♯

The fun keyword is superfluous, as is preppending Seq or List to higher order functions. Chaining with the |> operator is delightful.

★★★★☆ 
type Employee = {name:string; salary:int; age:int}

let employees = [
    {name="Adam"; salary=2000; age=20};
    {name="Bob"; salary=3000; age=50};
    {name="Clair"; salary=5000; age=60};
]

employees
    |> Seq.filter (fun e -> e.age > 40)
    |> Seq.map(fun e -> e.salary)
    |> Seq.sum
